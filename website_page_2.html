Database Normalization project

Project Introduction

This project is meant to explore the concept of database normalization using normal form. Normal form is tier of 7 levels that each has more specific rules a database should follow in order to be able to say it is in normal form. This set of rules ensures that the data in the table is well organized and ready for usage.

Takeaway

The main purpose of this project was to properly set up and organize a database. This is important because if a database is unable to follow normal form, it will struggle or fail to add or remove data into its tables. By following normal form, a database is fully functioning and ready to be deployed and used by whomever, while it is also easy to read and manipulate.
